The Problem

Most voice assistants are glorified search interfaces — they answer questions but can't take actions. They lack persistent memory, can't execute multi-step tasks, and hallucinate frequently when asked to reason about real-world tools.

Users need an AI that doesn't just talk — it acts. An agent that remembers context across sessions, can call real APIs, and produces reliable outputs even in production environments.

Technologies Used

Built with a focus on reliability, low latency, and production-grade agentic behavior.

LiveKit

Real-time voice communication with sub-300ms latency. WebRTC-based infrastructure for low-latency audio streaming and agentic response delivery.

Gemini

Google's multimodal model for natural language understanding, tool call generation, and context-aware reasoning with function calling support.

ChromaDB

Persistent semantic memory layer. Stores conversation embeddings for long-term context retention and retrieval-augmented generation.

Challenges Solved

Real-Time Voice Latency

Problem:

Voice AI must respond in under 1 second. Traditional request-response patterns create unacceptable latency for real-time conversation.

Solution:

Streaming responses with LiveKit's WebRTC infrastructure. Audio chunks are sent as soon as tokens are generated, achieving ~400ms perceived latency.

Memory Deduplication

Problem:

Without deduplication, the semantic memory grows unbounded and retrieval quality degrades over time.

Solution:

Implemented cosine similarity thresholding before embedding new memories. Only store if similarity score is below 0.85, keeping memory concise and retrieval relevant.

100% Test Coverage

Problem:

Agentic systems are notoriously hard to test. Tool calls can have side effects, and LLMs produce non-deterministic outputs.

Solution:

Dependency injection for all external services. Mock LLM responses for deterministic testing. pytest + pytest-asyncio for async tool execution tests.

What I Learned

Agentic Architecture Patterns

Building reliable agents requires careful separation of planning, execution, and verification. The planner-executor-verifier pattern is essential for complex task decomposition.

Streaming Audio Pipelines

WebRTC and streaming TTS/STT require careful buffer management. Backpressure handling and chunked delivery are critical for maintaining conversational flow.

Production AI Reliability

LLMs fail silently in production. Structured outputs, 2-pass correction, and extensive error handling are non-negotiable for user-facing AI products.

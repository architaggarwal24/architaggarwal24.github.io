Morphie-Bot

A dual-mode AI design copilot that separates conversation from structured design generation — switching on command, with strict output validation.

Flask Mistral AI Python JSON Schema

View on GitHub ↗
Morphie-Bot Interface

The Problem

Designers and developers waste hours translating vague briefs ("make it look modern and dark") into concrete color palettes and layout decisions. Asking an LLM for design suggestions produces walls of text — not actionable, frontend-safe outputs.

Generic chatbots also can't separate "I want to talk about design" from "give me a JSON palette I can drop into my CSS". The mode confusion wastes tokens and produces unusable outputs.

Technologies Used

Built for structured output reliability and clean mode separation.

Mistral AI

Mode-controlled system prompts switch the model's behavior between free-form conversation and strict structured JSON generation — same model, completely different output contract.

JSON Schema Enforcement

Every Design Mode response is validated against a strict schema before being returned to the frontend. Malformed outputs are caught and handled gracefully — never passed raw to the UI.

Flask + Session Memory

Server-side session memory maintains conversation history across mode switches — so context isn't lost when the user toggles between chat and design generation.

Challenges Solved

Mode Separation

Problem:

A single LLM can't reliably produce both casual conversation and strict JSON in the same session without the modes bleeding into each other.

Solution:

Separate system prompts injected per-mode with explicit output contracts. Mode is tracked server-side and enforced on every request — never trusted from the client.

Frontend-Safe Output

Problem:

LLMs occasionally wrap JSON in markdown code fences or add prose explanations — breaking downstream parsing.

Solution:

Post-processing strips markdown wrappers before validation. If the cleaned output still fails JSON parse, the error is caught and a fallback response is returned without crashing the UI.

Visual Color Presentation

Problem:

Hex codes in a text response are meaningless without visual context — users can't evaluate a palette they can't see.

Solution:

Frontend renders actual color swatches from the returned JSON in real time — users see the palette immediately without copy-pasting hex codes into a color picker.

What I Learned

Structured Output Foundations

This was my first project enforcing strict JSON schema validation on LLM outputs — the techniques I developed here (strip → parse → validate → fallback) became a core pattern in every subsequent project.

Mode-Controlled Behavior

System prompt engineering for behavioral mode switching is a lightweight alternative to fine-tuning. The key insight: the mode switch must happen server-side — never trust the client to declare which mode it's in.

Design Feedback Loops

Users want to iterate on designs conversationally — "make it darker", "add a secondary accent". Supporting iterative refinement requires maintaining design state across turns, not just generating on-demand.

The Problem

Job seekers spend hours tailoring resumes to each application, often without knowing if their bullet points actually match what recruiters are looking for. ATS systems reject applications based on keyword mismatches, and humans have no objective way to measure resume quality.

Existing solutions either use simple keyword matching (easily fooled by synonyms) or rely on expensive resume reviewers. Job seekers need an AI-powered tool that understands context, scores relevance, and provides actionable feedback.

Technologies Used

Built for reliability, performance, and a seamless user experience from PDF upload to LaTeX PDF generation.

Claude API

Structured JSON generation for resume scoring. 2-pass correction ensures valid outputs even with complex bullet point parsing requirements.

Supabase

PostgreSQL-backed user authentication, resume storage, and usage tracking. Row-level security ensures user data isolation.

Stripe

Usage-based billing for API calls. Webhook handling for subscription lifecycle management and invoice generation.

Challenges Solved

PDF Parsing Reliability

Problem:

PDF parsing libraries produce inconsistent results across different resume formats and encodings.

Solution:

Using pdfminer.six with preprocessing pipeline. OCR fallback for scanned documents. Validation layer ensures parsed content meets minimum quality thresholds.

LLM JSON Reliability

Problem:

Even with Claude, JSON outputs can be malformed or contain fields that break downstream processing.

Solution:

2-pass correction pattern: generate, validate, fix. JSON schema validation with clear error messages for invalid outputs. Pydantic models enforce type safety.

LaTeX PDF Generation

Problem:

Converting AI-improved bullet points back to a polished PDF while preserving the user's original styling choices.

Solution:

Template-based LaTeX generation with customization options. Jinja2 templating for flexible layouts. Compiled via XeLaTeX for Unicode support.

What I Learned

Full-Stack AI Architecture

Building AI products requires thinking across the entire stack — from PDF parsing to Stripe webhooks. Each layer has different reliability requirements.

Structured Output Enforcement

LLM outputs are never guaranteed to be valid. 2-pass correction, schema validation, and error recovery are essential for production AI systems.

User Trust in AI Tools

Users need to understand why the AI suggests changes. Explainable scoring and actionable feedback build trust more than opaque quality scores.

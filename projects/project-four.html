Autonomous Job Application Agent

An autonomous AI agent that hunts jobs, scores them against your resume, writes tailored cover letters, and applies — so you don't have to.

The Problem

Applying to jobs is a full-time job. The average job seeker spends 11 hours per week on applications — searching boards, tailoring cover letters, filling forms — for a process with a <3% response rate. Most of that time is pure repetition.

Generic applications don't work. Recruiters discard untailored cover letters instantly. But writing a personalized cover letter for every application doesn't scale. The solution is an agent that does both: high-volume search + per-application personalization via LLMs.

Technologies Used

Built for reliability, multi-platform coverage, and safe autonomous operation.

Playwright

Headless browser automation for scraping LinkedIn, Indeed, and Naukri. Human-mimicking delays and randomized interaction patterns prevent bot detection.

LangChain + Multi-LLM

Supports OpenAI, Mistral, and Anthropic for cover letter generation. Model is swappable via config — optimizing for cost vs quality per run.

TF-IDF Scoring + SQLite

TF-IDF relevance matching scores each listing against your parsed resume. SQLite deduplication ensures the agent never applies to the same role twice across daily runs.

Challenges Solved

Bot Detection

Problem:

LinkedIn and Indeed aggressively block automated scrapers with fingerprinting, CAPTCHAs, and behavioral analysis.

Solution:

Human-mimicking delays, randomized mouse paths via Playwright, and LinkedIn kept to scrape-only (no auto-apply) to protect account safety.

Cover Letter Quality

Problem:

Generic AI cover letters are worse than no cover letter — they signal the applicant didn't read the JD.

Solution:

Structured prompting injects both the parsed resume and full JD into the LLM context. Output is validated against minimum quality thresholds before submission.

Pipeline Reliability

Problem:

A single browser timeout or LLM error mid-pipeline would crash the entire daily run.

Solution:

Full error isolation per application — if one fails, the pipeline continues to the next. Dry-run mode enabled by default. Daily CLI dashboard reports success/failure counts per platform.

What I Learned

Safety-First Agent Design

Autonomous agents that interact with the real world need explicit safety defaults. Dry-run mode, rate limiting, and account-risk awareness are not optional features — they're core to responsible agent design.

Multi-LLM Architecture

Designing for LLM provider swap-ability from day one (via config, not code changes) is worth the initial overhead — it gives flexibility to optimize cost vs quality per use case.

Relevance Scoring Trade-offs

TF-IDF is fast and interpretable but misses semantic similarity. The next iteration would layer a sentence-transformer reranker on top to catch synonym matches that TF-IDF drops.
